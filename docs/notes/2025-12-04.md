## Demos
- Run `docker compose -f docker/api/compose.yml up -d` then `curl http://localhost:8080/health/ready` for API.
- CLI ingest: `docker run --rm -v $(pwd)/data:/data ghcr.io/ganesh47/sqlite-meta-cli:dev --help` then use sample datasets from `packages/cli/tests/fixtures`.
- Connector forwarder: `docker run --rm -p 8082:8082 ghcr.io/ganesh47/sqlite-meta-connector:dev` and POST events to `http://localhost:8082/events`.

## Performance & Telemetry
- CLI batching target: <5s latency per batch, RSS budget <256MB (logged via Pino serializers).
- Connector heartbeat emits status/latency/throughput; Prometheus metrics exposed on 8082 when enabled.
- No formal load test this cut; plan k6/Locust smoke in next demo.
